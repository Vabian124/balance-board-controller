## Summary

Created **Balance Board Controller**: a .NET 8 WPF rewrite of WiiBalanceWalker with vJoy game-controller output, modern dark UI, setup wizard, debug suite, file logging, and safe feeder-process cleanup. Published to `Vabian124/balance-board-controller` on GitHub.

## What was done

### Application (`src/BalanceBoard.App/`)
- WPF app with Dashboard + Debug Suite tabs
- `MainWindow` — live balance visual, connection/vJoy status pills, calibration, sensitivity
- `SetupWizardWindow` — prerequisites, pairing help, connect, tare, game preset
- `BalanceBoardVisual` control — real-time balance dot
- `ModernTheme.xaml` — dark theme (cards, pills, buttons)
- `App.xaml.cs` — single-instance mutex, feeder cleanup on startup, graceful shutdown

### Core (`src/BalanceBoard.Core/`)
- `BalanceBoardConnection` — WiimoteLib HID connect/read/tare
- `BalanceProcessor` — tare, center offset, deadzone, triggers, vJoy axis mapping
- `BalanceBoardSession` — 50 ms poll orchestrator
- `VJoyController` / `VJoyDiagnostics` — acquire, update, relinquish vJoy
- `InputSimulator` — keyboard/mouse via SendInput
- `FeederProcessCleanup` — kill stale `BalanceBoardApp`, `WiiBalanceWalker`, `WBBGUI`
- `SettingsStore` — JSON settings in `%AppData%\BalanceBoardApp\`
- `FileLogService` — session logs (gitignored)
- `DiagnosticsReport` — unified health check

### Tooling & libs
- `tools/Validate/` — CLI health check + vJoy axis sweep
- `libs/x64/` — WiimoteLib + vJoy DLLs
- `BalanceBoard.sln` — App + Core

### Docs & legal
- `README.md`, `LICENSE` (MIT), `THIRD_PARTY_NOTICES.md`
- `.gitignore` — logs, baseline, reference excluded

### Legacy reference
- `WiiBalanceWalker/` — original v0.5 source (MS-PL), retargeted .NET 4.8.1 for reference builds

### Bugs fixed during this session (before commit)
- Load settings before `InitializeComponent()` (slider crash)
- `_uiReady` guard on UI saves
- vJoy BUSY — feeder cleanup + single instance
- `WiimoteNotFoundException` on discovery — try/catch
- Duplicate sensor reads — poll only, no `ReadingReceived` handler
